# Local Hybrid Search — 運用ガードレール

このファイルは `local_hybrid_search.ts` を操作する際の**ガードレール**（巻き戻し禁止ルール）を定義する。
変更には人間の明示的な合意が要る。

---

## 書き込みゲート

- 小型モデルに保存層への **自動書き込みをさせない**。
  `put()` の呼び出しは、人間のアクションか明示的に定義されたルールによってのみトリガーする。
- LLM が自律的に知識を追加・上書きすることは禁止。

## 鮮度と忘却

- **decay / 忘却曲線を実装しない。**
  鮮度は `last_confirmed_at` と明示的な `addEvidence()` による再確認で表現する。
- 時間経過だけで知識を自動的に stale 化しない。

## フィルタリングの方向

- **取り込み時にフィルタする**（retention でフィルタしない）。
  DB に入れるかどうかは入口で判断する。一度入ったものを自動削除しない。

## slug の不変性

- `slug` は概念の不変ハンドル。
  表示名・content は `put()` で更新できるが、slug 自体のリネームは行わない。
- Wiki リンクは slug を指す。slug を変えるとリンクが壊れる。

## 埋め込みモデル

- **モデル名・次元・dtype を勝手に変えない。**
  現在: `ku-nlp/ruri-v3-310m` / 768 次元 / `q8`。
  変更はすべてのベクトルの再計算を要求する（一方通行のドア）。
  変更する場合は設計メモの「再検討トリガー」に従い、明示的な合意を得る。

## スキーマ不変条件

- 「1 slug に live は高々 1 件」を維持する（部分ユニークインデックス）。
- FTS / vec テーブルの `rowid` は `knowledge.id` と同一であることを維持する。
- スキーマ変更時もこれらの不変条件を壊さない。

## 検索品質

- ベクトル量子化（int8 等）の導入前に、品質ベースラインとの比較を行う。
- 小型モデルへの切り替えは、実データで品質が落ちないことを測定してから。
